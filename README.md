## 1. Постановка задачи

В рамках работы проводится сравнительный анализ стандартных и специализированных алгоритмов сортировки строк.

Цель исследования — оценить влияние посимвольных сравнений и использования префиксных структур на производительность алгоритмов.

В ходе экспериментов рассматриваются как классические алгоритмы сортировки, так и их модификации, оптимизированные под работу со строками.

---

## 2. Реализованные алгоритмы

В работе реализованы следующие алгоритмы:

### Классические алгоритмы:
- QuickSort
- MergeSort

### Специализированные алгоритмы сортировки строк:
- Тернарный String QuickSort
- String MergeSort с использованием LCP
- MSD Radix Sort
- MSD Radix Sort с переключением на String QuickSort

Все реализации объединены в рамках класса `StringSortTester`.

---

## 3. Генерация тестовых данных

Для формирования входных данных используется класс `StringGenerator`.

### Параметры генерации:
- Длина строк: от 10 до 200 символов
- Размер массива: от 100 до 3000 элементов (шаг 100)
- Алфавит: 74 символа (латинские буквы, цифры и специальные символы)

### Типы входных данных:
- случайные массивы
- отсортированные массивы
- обратно отсортированные массивы
- почти отсортированные массивы
- массивы с частично совпадающими префиксами

Для ускорения экспериментов сначала генерируется массив максимального размера (3000 элементов), а дальше используются его подмассивы.

---

## 4. Методика измерений

Для оценки эффективности алгоритмов использовались две метрики:

- время выполнения (мкс)
- количество посимвольных сравнений

### Особенности проведения экспериментов:
- многократное выполнение каждого теста
- усреднение результатов
- фиксация seed генератора для воспроизводимости

---

## 5. Обработка результатов

Обработка и визуализация результатов выполняется с использованием Python-скрипта:

- `plot_results.py`

После `main.cpp` данные сохраняются в csv, после чего строятся графики зависимости времени выполнения и количества сравнений от размера входного массива.

---

## 6. Результаты экспериментов

Построены 6 графиков (3 со шкалой времени выполнения + 3 со шкалой количества сделанных сравнений), соответствующих различным типам входных данных:

1. случайные массивы
3. обратно отсортированные массивы
4. почти отсортированные массивы

На каждом графике представлены все рассматриваемые алгоритмы.

---

## 7. Анализ результатов

На всех типах входных данных классический QuickSort показывает наибольшее количество посимвольных сравнений и сильнее зависит от структуры массива. MergeSort демонстрирует более стабильное поведение и меньший рост сравнений, однако по времени работы все равно уступает специализированным алгоритмам.

String QuickSort быстрее выполняет сортировку, при этом делая меньше проверок, по сравнению с обычным QuickSort за счёт посимвольной обработки строк, но всё ещё остаётся тяжелее некоторых адаптированных методов. String MergeSort же благодаря использованию LCP заметно уменьшает число сравнений, но на практике показывает довольно большое время выполнения из-за дополнительных накладных расходов реализации.

Наилучшие результаты практически на всех графиках демонстрируют MSD Radix Sort и MSD Radix Sort With Switch. Эти алгоритмы выполняют минимальное число посимвольных сравнений и показывают наименьшее время работы на случайных, обратно отсортированных и почти отсортированных массивах. Гибридная версия с переключением на String QuickSort, естественно, дополнительно улучшает производительность на небольших подмассивах.

Таким образом, полученные результаты подтверждают, что специализированные алгоритмы сортировки строк оказываются эффективнее классических алгоритмов сравнения как по времени работы, так и по количеству посимвольных сравнений.

## 8. Вывод

В ходе исследования установлено, что специализированные алгоритмы сортировки строк явно эффективнее классических особенно при больших объемах данных. MSD Radix Sort показал себя наилучшим сортирующим алгоритмом. Так же было выяснено, что использование LCP и побуквенной обработки уменьшает количество сравнений, а гибридные подходы, как и ожидалось, обеспечивают более стабильную производительность на различных типах входных данных.
